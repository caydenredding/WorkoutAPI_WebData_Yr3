# Tables
## Initial Iteration
The initial iteration has 4 tables which link all users, workout logs, exercise logs, and exercises
This can be improved however by adding more tables like by giving each muscle an ID rather than storing as a string to save on space.
Also may make finding analytics easier later, which may be good to include before then.
Also opens potential for a set log as a limitiation of the current version is that all of the sets must be of the same weight (although it should be possible to use separate sets as separate exercises as the indiviudaul ID should be OK)
However this solution is not great, and more of a workaround so a future addition should be the sets table.
